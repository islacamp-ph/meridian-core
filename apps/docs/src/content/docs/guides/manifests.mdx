---
title: Ecosystem Manifests
description: Describe known contracts for richer dependency mapping
---

An ecosystem manifest is a JSON file describing known contracts in your deployment. FIELD and GRAVITY use it to enrich dependency graphs, blast-radius scoring, and affected-user counts.

## Schema

```json
{
  "name": "my-ecosystem",
  "version": "1.0.0",
  "contracts": [
    {
      "name": "token-vault",
      "address": "CABC...XYZ",
      "network": "testnet",
      "dependencies": ["CDEF...UVW"],
      "active_users": 4200,
      "criticality": "HIGH",
      "role": "vault"
    }
  ]
}
```

## Fields

| Field | Required | Description |
|---|---|---|
| `name` | Yes | Ecosystem identifier |
| `version` | Yes | Manifest semver |
| `contracts[].name` | Yes | Human-readable contract name |
| `contracts[].address` | Yes | Contract address (C... strkey) |
| `contracts[].network` | Yes | `mainnet` or `testnet` |
| `contracts[].dependencies` | No | Other contract addresses this contract calls |
| `contracts[].active_users` | No | Estimated active users (for blast radius) |
| `contracts[].criticality` | No | `HIGH`, `MEDIUM`, or `LOW` |
| `contracts[].role` | No | Free-text role description |
| `contracts[].expected_wasm_hash` | No | 64-char hex SHA-256 of expected Wasm (upgrade drift) |
| `contracts[].audit_status` | No | `audited`, `unaudited`, or `unknown` |
| `contracts[].deployed_at` | No | ISO-8601 deployment date |
| `contracts[].deployed_ledger` | No | First-seen / deployment ledger |
| `contracts[].upgradeable` | No | Whether bytecode can be upgraded |
| `contracts[].reputation_score` | No | Manifest reputation 0–100 |

Counterparty fields feed GRAVITY (unaudited / upgradeable / low reputation factors), execution-graph node metadata, and the `untrusted_counterparty` policy rule. There is no external reputation oracle — scores are manifesto-supplied.

## Community library

Shared manifests live in the repository `manifests/` directory. Validate them with:

```bash
meridian manifest validate manifests/<ecosystem>/manifest.json
```

## Scaffold a new manifest

```bash
meridian init --name my-ecosystem --network testnet
```
